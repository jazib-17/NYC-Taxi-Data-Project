For this project, I explored the NYC Taxi trip dataset from August 2025 to analyze trip patterns, calculate trip durations, and generate summary statistics. I designed the workflow to work easily on Windows, using either Pandas or PySpark, depending on the size of the dataset and available memory. The goal was to make the data cleaning, processing, and analysis reproducible and straightforward for anyone following along.

I start by loading the raw data either as a CSV using Pandas or as a Parquet file using PySpark. For very large datasets that might not fit in memory, I either process the data in chunks with Pandas or use PySpark to handle the full dataset efficiently. Once the data is loaded, I compute a new column called trip_duration, calculated from the pickup and dropoff timestamps. I also calculate total trip amounts including tips, and I clean the data by removing invalid or incomplete rows to ensure accuracy in the analysis.

After cleaning, I save the processed data to a separate folder for organization. On Windows, saving as CSV is the simplest option because it avoids the need for Hadoop or winutils setup, while saving as Parquet requires PySpark and a proper Hadoop environment. I then compute summary statistics such as average fare amounts and average total amounts including tips, rounding results to two decimal places for readability.

For Windows users, the simplest workflow is to use Pandas only. If PySpark is desired, it requires either a full Hadoop installation or placing winutils.exe in the appropriate folder and setting the HADOOP_HOME environment variable. For large datasets over one gigabyte, Pandas may run into memory limits, so chunked processing or PySpark is recommended. For portfolio or demo purposes, I often work with a sample of the data, around 50,000 rows, to make the pipeline faster to run. Throughout the project, I keep raw data separate in a raw folder and save all cleaned and processed outputs in a processed folder to stay organized.

This project documents my approach to cleaning, processing, and analyzing NYC Taxi data on Windows, along with practical tips for handling large datasets and deciding whether to use Pandas or PySpark depending on the environment and file size. The workflow is designed to be reproducible, efficient, and beginner-friendly while still showcasing meaningful data analysis techniques.
